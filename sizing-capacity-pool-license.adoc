= ONTAP Select 容量池授權容量規劃注意事項
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
容量池授權的授權容量以完整 TB 為單位，最小值為 2 TB。授權容量的計算方法是將可用空間量乘以開銷係數。單節點叢集和多節點（基於 HA 配對）叢集的開銷係數有所不同：

* 單節點叢集的開銷係數為 1.13
* 多節點叢集的開銷係數為 2.67


您需要根據可用空間計算授權容量，以確保您的部署有足夠的容量。

[cols="25,30,45"]
|===
| 說明 | 單節點 | HA 配對或 HA 配對與單節點的混合 


| 可用空間 | 20 TB | 單節點叢集 10 TB HA 對 20 TB 


| 授權空間 | 23 TB（20 x 1.13 = 22.6，向上取整到下一個整數 TB） | 54 TB（20 x 2.67 = 53.4，向上取整到下一個整數 TB） 


| 每個節點的最大可用空間 *註*：每個節點或 HA 配對的最大管理空間為 400 TB | 353 TB（400 / 1.13）  a| 
叢集中的每個單一節點：

149 TB（400 / 2.67）

叢集中每個 HA 配對：

298 TB （2 x 149）

|===
