= ONTAP Select Capacity Tiers 授權容量規劃注意事項
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
容量層級授權是基於每個 ONTAP Select 節點所管理的底層磁碟容量，而非節點、HA 配對或叢集的可用容量。每個容量層級授權都分配給一個特定的節點序號，並對應一個指定的容量，且需要簽訂相應的支援合約。

容量層級授權的授權容量以完整的 TiB 為單位，最小值為 1 TiB。授權容量的計算方法是將可用空間乘以開銷係數。單節點叢集和多節點（基於 HA 配對）叢集的開銷係數有所不同：

* 單節點叢集的開銷係數為 1.13
* 多節點叢集的開銷係數為 2.67


根據可用空間計算授權容量，以確保您的部署有足夠的容量。

[cols="25,30,45"]
|===
| 說明 | 單節點 | HA配對 


| 可用空間 | 10 TiB | 每個節點 5 TiB，HA 配對 10 TiB 


| 授權空間 | 12 TiB（10 x 1.13 = 11.3，向上取整到下一個整數 TiB） | 兩個 14 TiB 授權（5 x 2.67 = 13.35，向上取整到下一個完整 TiB） 


| 每個節點的最大可用空間 *注意*：每個節點或 HA 對的最大管理空間為 400 TiB | 353 TiB（400 / 1.13）  a| 
HA 對中的每個節點：

* 74 TiB ［（400 / 2）/ 2.67］


在多節點叢集中，每個 HA 對：

* 雙節點：148 TiB（2 x 74）
* 四節點：296 TiB（4 x 74）
* 六節點：444 TiB（6 x 74）
* 八節點：592 TiB（8 x 74）


|===
